Build Github Pages with Pelican

Posted on 周日 01 一月 2017 in Git

0. Pre-knowledge.

  • Git/Github/Github Pages
  • Python/Pip/Pelican
  • Markdown/reStructuredText

1. Installation.

Install pip
  • wget https://raw.github.com/pypa/pip/master/contrib/get-pip.py [1]
  • python get-pip.py
If needed, upgrade 'pip':
python -m pip install --upgrade pip

Install pelican
  • pip install pelican

2. Pelican-Quickstart.

Create the root directory of your blog and quickstart:
  • mkdir blog
  • cd blog
  • pelican-quickstart
Welcome to pelican-quickstart v3.7.0.

This script will help you create a new Pelican-based website.

Please answer the following questions so this script can generate the files needed by Pelican.

> Where do you want to create your new web site? [.]
> What will be the title of this web site? Anything-Is-Possible

Create sub-directories under ./blog/content/
  • mkdir imgs pages posts
Now the file/dir structure of './blog' should looks like:
.
├── content
│   ├── imgs
│   ├── pages
│   │      └── (pages)
│   │
│   └── posts
│          └── (categories/articles)
│
├── develop_server.sh
├── fabfile.py
├── Makefile
├── output
├── pelicanconf.py
└── publishconf.py
Get the pelican-themes:
  • git clone https://github.com/getpelican/pelican-themes.git [2]

Select your favorite theme, here I choosed flex which I think wonderful and made some changes with 'aside' and 'css'.

So I named this modified version - Relex.

Now change the pelicanconfig.py, add below lines.
  • vi pelicanconfig.py
THEME = 'pelican-themes/relex'

STATIC_PATHS = ['imgs']

FAVICON = '/imgs/favicon.ico'

LOAD_CONTENT_CACHE = False
USE_FOLDER_AS_CATEGORY = True

ARTICLE_PATHS = 'posts'
ARTICLE_URL = 'posts/{category}/{slug}.html'
ARTICLE_SAVE_AS = ARTICLE_URL

3. Write the 1st Article

Execute below steps under 'blog'

Create category:
  • mkdir -p content/post/your-1st-category
Create article:
  • vi ./blog/content/posts/your-1st-art.rst
=========================
Your 1st Awesome Article
=========================

:date: 2017-1-1
:modified: 2017-1-1
:category: your-1st-category
:tags: Awesome
:slug: your-1st-art
:author: ylnz

Anything is possible if you try hard enough.
Generate the output:
  • make html
Start server:
  • devestack.sh start

Now, open your web browser, locate 'http://localhost:8000'

4. Publish to Github

Create the repo on Github with name '<your-github-name>.github.io'

cd ./blog/output/

git init
git add .
git remote add origin https://github.com/JustWanna/justwanna.github.io
git pull origin master
git commit -m '1st commit'
git push origin master

Once finished, access 'http://YOUR-USERNAME.github.io'

5. Apex Domain Binding

To enable the apex domain(such as www.example.com or example.com), here for example I use 'gitcc.cc'.

  • Check the IP address for your github pages via dig cmd:
# dig YOUR-USERNAME.github.io +noall +answer
...
xxxxxx.github.io.       3600    IN      CNAME   github.map.fastly.net.
github.map.fastly.net.  30      IN      A       151.101.xx.xxx

  • Create an 'A' record on your DNS provider's control board to points to the GitHub Pages server [3].

    Simply refer to Github Pages' official guide: Under your repository name, click Settings, Under "Custom domain", add or remove your custom domain and click Save [4].
